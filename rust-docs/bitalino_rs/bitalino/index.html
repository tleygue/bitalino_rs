BITalino device driver for data acquisition.

This module provides the core driver functionality to communicate with BITalino biosignal acquisition devices over serial or RFCOMM connections.

§Timing and Synchronization

The BITalino device has an internal crystal oscillator that controls sampling. When you call start(), the device begins streaming at the configured rate (1, 10, 100, or 1000 Hz) with high precision (~20 ppm crystal accuracy).

Important timing considerations:

  • The device does NOT send timestamps - timing must be reconstructed on the host
  • Bluetooth introduces variable latency (typically 10-50ms)
  • Data may arrive in bursts due to Bluetooth buffering
  • The 4-bit sequence number (0-15) allows detection of dropped frames
